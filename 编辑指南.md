# SLDX_Ambition_Website 编辑指南

## 📖 前言

本指南面向没有编程基础的团队成员，教你如何更新战队网站内容。

**重要说明**：
- 所有修改都需要通过 **Pull Request (PR)** 提交给管理员审核
- 审核通过后会由管理员合并到主分支，网站会自动更新
- 不要直接修改主分支，需要通过 PR 流程

---

## 🚀 快速开始（三种方式）

### 方式一：在线编辑（最简单，推荐新手）

适合快速修改文字内容，无需安装任何软件。

#### 步骤：

1. **打开要编辑的页面**
   - 访问网站：https://sldx-ambition.top
   - 找到要编辑的页面

2. **点击"在 GitHub 上编辑"按钮**
   - 在页面底部找到编辑按钮
   - 首次使用会提示你 Fork 仓库到个人账号

3. **编辑内容**
   - 在 GitHub 编辑器中修改内容
   - 支持 Markdown 语法（见下方教程）

4. **提交更改**
   - 滚动到页面底部
   - 填写修改说明（必填）
   - 选择 **"Create a new branch"**
   - 点击 **"Propose changes"**

5. **创建 Pull Request**
   - 点击 **"Create pull request"**
   - 等待管理员审核
   - 审核通过并合并后，网站会自动更新（约 1-2 分钟）

### 方式二：使用 VS Code 编辑（推荐）

适合大量修改或批量更新。

#### 1. 安装必要软件

- **VS Code**：https://code.visualstudio.com/
- **Git**：https://git-scm.com/zh-cn
- 安装 VS Code 插件：`Markdown Preview Enhanced`

#### 2. Fork 仓库

1. 打开 https://github.com/SYLU-Ambition/SLDX_Ambition_Website
2. 点击右上角 **Fork** 按钮
3. 复制你的仓库地址

#### 3. 克隆仓库到本地

```bash
# 选择一个文件夹
cd ~/Documents  # Mac/Linux
# 或
cd D:\Projects  # Windows

# 克隆你的 fork 仓库
git clone https://github.com/你的用户名/SLDX_Ambition_Website.git

# 进入项目目录
cd SLDX_Ambition_Website
```

#### 4. 编辑内容

```bash
# 用 VS Code 打开项目
code .
```

- 所有页面在 `doc/` 目录下
- 编辑 `.md` 文件
- 按 `Ctrl+Shift+V` 预览效果

#### 5. 提交并发起 Pull Request

```bash
# 添加修改
git add .

# 提交更改
git commit -m "修改说明：更新了 XX 内容"

# 推送到你的仓库
git push
```

然后在 GitHub 上创建 Pull Request：

1. 打开你的仓库页面：`https://github.com/你的用户名/SLDX_Ambition_Website`
2. 点击 **"Contribute"** → **"Open pull request"**
3. 选择 base 仓库为 `SYLU-Ambition/SLDX_Ambition_Website:main`
4. 填写 PR 说明，点击 **"Create pull request"**

### 方式三：使用 GitHub Desktop（图形化工具）

适合不熟悉命令行的同学。

1. 下载安装：https://desktop.github.com/
2. 登录后克隆你的 fork 仓库
3. 在本地编辑文件
4. 在 GitHub Desktop 中提交并推送
5. 在 GitHub 上创建 Pull Request

## 🔄 Pull Request (PR) 完整流程

### 什么是 Pull Request？

Pull Request 是你向项目提出"我想修改这些内容，请审核并合并"的请求。

### 完整流程：

```
1. Fork 仓库 → 复制一份到你的账号
2. 修改内容 → 编辑文件或代码
3. 提交更改 → 保存到你的仓库
4. 创建 PR → 请求管理员审核
5. 等待审核 → 管理员检查修改
6. 合并 PR → 修改被接受，网站自动更新
```

### PR 被拒绝怎么办？

- 管理员可能会提出修改建议
- 根据评论修改后重新提交即可
- 这是正常的协作流程，不要担心

## ❓ 常见问题

### Q: 修改后网站为什么没有更新？

A: 需要等待管理员审核并合并 Pull Request，合并后约 1-2 分钟自动更新。

### Q: 如何查看我的 PR 状态？

A: 在 GitHub 仓库的 **Pull requests** 标签页查看。

### Q: 多人同时修改会冲突吗？

A: 如果修改不同文件不会冲突。如果修改同一文件，后提交的人需要解决冲突。

### Q: 如何撤销已提交的 PR？

A: 在你的 PR 页面底部点击 **"Close pull request"**。

### Q: 可以在本地预览效果吗？

A: 可以，需要安装 Node.js 和 pnpm：
```bash
pnpm install
pnpm dev
```
